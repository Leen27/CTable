## 推荐方案

**样式文件应该放在：`packages/table-core/src/styles/` 目录下**

### 具体建议：

1. **创建样式目录结构**：

```
packages/table-core/src/styles/
├── index.css           # 主样式文件，导出所有样式
├── table.css           # 表格基础样式
├── header.css          # 表头样式
├── body.css            # 表格主体样式
├── footer.css          # 表脚样式
├── cell.css            # 单元格样式
├── row.css             # 行样式
├── themes/             # 主题样式
│   ├── default.css
│   ├── dark.css
│   └── light.css
└── components/         # 组件特定样式
    ├── pagination.css
    ├── filtering.css
    └── sorting.css
```

2. **为什么这样设计**：
   - **模块化**：每个功能模块有独立的样式文件，便于维护
   - **可扩展**：支持多主题和组件级样式定制
   - **清晰结构**：与现有的 `features/` 目录结构对应
   - **易于导入**：可以按需导入特定样式模块

3. **在代码中使用**：
   - 在 [`RenderGrid.ts`](packages/table-core/src/features/RenderGrid.ts:110) 中，可以看到已经使用了内联样式
   - 可以将这些内联样式提取到对应的 CSS 文件中
   - 使用 CSS 类名替代内联样式，提高可维护性

4. **构建配置**：
   - 由于项目使用 Vite 构建，CSS 文件会被自动处理
   - 可以在 [`index.ts`](packages/table-core/src/index.ts:1) 中导出样式文件路径
   - 用户可以选择性导入所需的样式模块

这种结构既保持了代码的模块化，又提供了良好的样式组织和管理能力。
